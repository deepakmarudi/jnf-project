# Team Setup

## Purpose

This file explains how all teammates should start working in this repository.

## Repository Rule

We use one shared GitHub repository.

- `main` = stable branch
- `develop` = shared working branch
- `feature/...` = personal task branches

## Team Workflow

Each teammate should:

1. Clone the shared repository
2. Switch to `develop`
3. Pull the latest `develop`
4. Create a personal `feature/...` branch
5. Do the assigned work only in that branch
6. Push the feature branch
7. Create a Pull Request into `develop`

## Important Rules

- Do not work directly on `main`
- Do not push directly to `develop`
- Do not commit `.env` or secrets
- Open Pull Requests only into `develop`

## Project Structure

- `docs/` = project documentation
- `frontend/` = Next.js frontend
- `backend/` = backend documentation and implementation planning
